📖 FAQ & Help — CAN Log Checker

What does this tool do?

CAN Log Checker is a lightweight validation tool for CAN bus log files. It checks whether key vehicle parameters (speed, RPM, brake, fuel, PTO, etc.) are present in the log and shows a pass/partial/fail result.

How do I load a file?

Click Browse or drag-and-drop a .csv file onto the header drop zone. Supported formats include IXXAT MiniMon, canAnalyser 3 Mini, and USB-to-CAN V2 compact exports.

What do the banner colours mean?

  • 🟢 Green (PASS) — All or nearly all priority data values found (0–1 missing).
  • 🟠 Orange (PARTIAL) — Some priority values are missing (2+ missing).
  • 🔴 Red (FAIL) — No decodable data found in the file.

What is "Priority Data"?

Priority Data shows the maximum observed values for 12 critical vehicle parameters: Speed, Engine RPM, Torque, Engine Load, Accelerator, Exhaust Brake, Fuel Rate, Fuel Used, Distance, Park Brake, Brake Pedal, and PTO State.

What does the timeline graph show?

Four traces: Engine RPM (cyan, left axis), Accelerator % (green, right axis), Brake Pedal % (red, right axis), and PTO (amber, stepped on/off). The X-axis uses actual timestamps when available.

What is the CAN Decode section?

A collapsible section showing all decoded PGNs in three tiers: Decoded PGNs (with full SPN data), Known PGNs — Hex Only (recognised PGN name but no SPN decoding), and Unknown / Proprietary (unrecognised PGN IDs).

Browser compatibility

Chrome 90+, Firefox 88+, Edge 90+, Safari 14+. All processing is client-side — no data is uploaded anywhere.

What makes a good CAN log?

A good log should have: valid J1939 CAN messages (29-bit extended IDs), a reasonable number of frames (1,000+), timestamps, and data from the FMS gateway (PGNs like F004, FEF1, FEF2, etc.). Short logs or logs from non-FMS ports may show missing data.

Why did my file show a FAIL result?

Common reasons: the file is not from a J1939/FMS CAN bus, the CSV format is not compatible, the log was recorded from the wrong bus (e.g., body builder bus instead of FMS gateway), or the file is corrupted/empty.

Is my data safe?

Yes. CAN Log Checker runs entirely in your browser — no data is uploaded to any server, no internet connection is required, and no cookies or tracking are used.

🔧 How CAN Data is Decoded

CAN Bus Basics

CAN (Controller Area Network) is a vehicle bus standard that allows electronic control units (ECUs) to communicate. Each CAN message has a 29-bit or 11-bit identifier and up to 8 bytes of data.

J1939 PGN Extraction

For J1939 (the heavy-duty vehicle standard), the 29-bit CAN ID is split into fields. The Parameter Group Number (PGN) is extracted from bits 8–25:

  • Right-shift the 29-bit ID by 8 bits to remove the Source Address.
  • If PDU Format (PF) byte < 240 → peer-to-peer (PGN uses PF byte only).
  • If PF ≥ 240 → broadcast (both PF and PS bytes form the PGN).

Example: CAN ID 0CF00400 → right-shift 8 → PF = F0 (≥240) → PGN = F004 (EEC1 — Engine RPM lives here!).

SPN Decoding

Each PGN contains SPNs (Suspect Parameter Numbers). Each SPN specifies: byte offset, bit length, resolution, offset, and optional labels. Formula: value = raw × resolution + offset.

Example: SPN 190 (Engine Speed) in PGN F004 — bytes 3–4, resolution 0.125 RPM/bit. If bytes = 0x1388 (5000), then RPM = 5000 × 0.125 = 625 RPM.

🔍 Troubleshooting

File won't load / nothing happens

Make sure the file is .csv or .txt format with columns for Time, Identifier (or ID), and Data. The tool looks for these column headers in the first 15 lines.

All values show as "—"

The log may not contain J1939/FMS data. Check that the CAN bus you recorded from is the FMS gateway, not a body builder, diagnostic, or infotainment bus.

FAIL result on a valid-looking file

The file may be from a non-J1939 bus (e.g., OBD-II passenger car, body builder, or proprietary bus). CAN Log Checker specifically validates J1939/FMS parameter groups.

Orange PARTIAL banner

Some vehicles don't broadcast all 12 priority parameters — this is normal. Electric vehicles won't have fuel data, and some ECU configurations omit parameters like exhaust brake or PTO.

📚 Glossary of Key Terms

  • CAN — Controller Area Network, a robust vehicle communication bus standard invented by Bosch.
  • J1939 — SAE standard for heavy-duty vehicle CAN communication (trucks, buses, construction equipment).
  • FMS — Fleet Management System, a common data interface agreed by European truck manufacturers for telematics access.
  • PGN — Parameter Group Number, identifies a group of related parameters in a J1939 message (e.g., F004 = Engine Controller).
  • SPN — Suspect Parameter Number, a specific data value within a PGN (e.g., SPN 190 = Engine Speed).
  • ECU — Electronic Control Unit, a computer module in the vehicle (engine ECU, brake ECU, transmission ECU, etc.).
  • PTO — Power Take-Off, an auxiliary drive that transfers engine power to external equipment (e.g., hydraulic pumps).
  • DTC — Diagnostic Trouble Code, a fault code stored by an ECU when a problem is detected.
  • Source Address (SA) — The last byte of the J1939 CAN ID, identifying which ECU sent the message (0–253).

🎉 Fun CAN Bus Facts

  • 🚗 CAN was invented by Robert Bosch GmbH in 1986 — originally to reduce wiring weight in Mercedes-Benz cars. A modern truck has over 2 km of wiring!
  • 📡 A CAN bus carries up to 2,000+ messages per second at 250 kbit/s — more data than a dial-up internet connection.
  • 🔢 J1939 defines over 8,500 SPNs. CAN Insight PRO decodes 200+ of the most important ones — the top 2.4%!
  • 🌍 The FMS standard is used by all major European truck makers — DAF, Iveco, MAN, Mercedes-Benz, Renault Trucks, Scania, and Volvo.
  • ⚡ Electric truck battery charge (SPN 5464) updates every 10 seconds. On a 300 kWh battery, 1% = enough energy to power a house for 3 hours!
  • 🔧 "Suspect Parameter Number" comes from engine diagnostics — every parameter is a "suspect" when something goes wrong. A detective lineup for your engine!
  • 📏 A CAN frame carries only 8 bytes — fewer characters than "Hello!!!" — yet encodes everything from engine speed to battery voltage.
  • 🏎️ CAN bus is also used in Formula 1 cars, aircraft, medical devices, elevators, and even coffee machines!
  • 🚛 A typical long-haul truck generates about 1 GB of CAN data per day — enough to fill a floppy disk every 90 seconds!
  • 🔌 CAN bus wires are twisted pairs — about 33 twists per metre to reject electromagnetic interference from the engine.
  • 📅 The first production car with CAN bus was the 1991 Mercedes-Benz W140 S-Class — replacing over 2 km of conventional wiring.
  • 🔋 Modern electric trucks can have up to 5 separate CAN buses — powertrain, chassis, body, infotainment, and battery management.
  • 🛡️ J1939 reserves special addresses: 254 is "null", 255 is global broadcast, and address 0 is typically the engine ECU.
  • ⏱️ The fastest classic CAN bit rate is 1 Mbit/s, but trucks use 250 kbit/s for reliability over longer cable runs.
  • 🚂 CAN bus is used in trains, tractors, ships, and even spacecraft — NASA's Mars rovers use CAN-based communication!
  • 🌐 CAN FD (Flexible Data-rate) carries 64 bytes per frame instead of 8 — 8× more data to meet modern vehicle needs.
