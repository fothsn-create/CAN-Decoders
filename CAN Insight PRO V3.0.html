📖 FAQ & Help

What file formats are supported?

CAN Insight PRO accepts .csv log files exported from IXXAT MiniMon, canAnalyser 3 Mini, USB-to-CAN V2 compact, and similar CAN bus logging tools. The file must contain columns for CAN ID (or Identifier) and Data bytes.

How do I load a file?

Click LOAD LOG CSV or drag-and-drop a CSV file onto the application. The file will be parsed automatically and all tabs will populate with decoded data.

What are the tabs?

  • Raw Data — Telemetry summary, per-PGN decoded parameters, and supported PGN list.
  • Timeline — Interactive time-series chart of RPM, accelerator, brake, PTO, gear, and all other available signals. Use the ◀ ▶ buttons to scroll and + − to zoom.
  • Tachograph — Remote Tachograph Download (RTD) data when available in the log.
  • Manual Decode — Enter a single CAN ID & data payload to decode any frame manually.

What does "Priority Data" mean?

Priority Data shows the maximum observed values for critical vehicle parameters (speed, RPM, torque, fuel, brake, PTO, etc.). A green banner means all priority values were found; orange means some are missing.

Why are some values shown as "—"?

A dash indicates that particular PGN/SPN was not present in the loaded log file. This is normal — not all vehicles broadcast every parameter.

How do the timeline zoom/scroll controls work?

Use the arrow pad buttons in the timeline toolbar: ◀ scroll left, ▶ scroll right, + zoom in, − zoom out, ⟲ reset. Pinch-to-zoom is also supported on touch devices.

Browser compatibility

Chrome 90+, Firefox 88+, Edge 90+, Safari 14+. All processing is client-side — no data is uploaded anywhere.

What export formats are available?

Timeline PNG — screenshot of the visible chart. Wide PNG — 3× width for pasting into reports. Excel — full timeline data and signal statistics in .xlsx format.

A browser-based tool for decoding and analysing heavy-duty vehicle CAN bus data using the J1939 / FMS (Fleet Management System) standard. No installation required — just open this HTML file in any modern browser.

🔒 Privacy: All data is processed entirely in your browser. Nothing is uploaded to any server.

Features

  • 64+ J1939/FMS PGN definitions with full SPN decoding
  • Real-time telemetry dashboard — RPM, speed, fuel, brake, accelerator, PTO, gear
  • Interactive timeline chart — multi-trace with zoom/scroll controls
  • Vehicle identification — automatic VIN and make detection
  • Tachograph data — RTD (Remote Tachograph Download) decoding
  • Manual decode — enter any CAN ID & data payload to decode a single frame
  • Export — PNG, wide PNG, and Excel export for timeline and data overview

Known Limitations

  • Supports J1939 extended (29-bit) CAN IDs only; standard 11-bit frames are listed but not decoded
  • CSV files must have columns for CAN ID (or Identifier) and Data bytes
  • Tested with IXXAT MiniMon, canAnalyser 3 Mini, and USB-to-CAN V2 compact CSV exports
  • No multi-frame (Transport Protocol) reassembly — each frame is decoded independently
  • All processing is client-side; very large files (>100 MB) may be slow on older hardware

🔧 How CAN Data is Decoded

CAN Bus Basics

CAN (Controller Area Network) is a vehicle bus standard that allows electronic control units (ECUs) to communicate. Each CAN message has a 29-bit or 11-bit identifier and up to 8 bytes of data.

J1939 PGN Extraction

For J1939 (the heavy-duty vehicle standard), the 29-bit CAN ID is split into fields. The Parameter Group Number (PGN) is extracted from bits 8–25 of the extended CAN ID:

  • The raw 29-bit ID is right-shifted by 8 bits to remove the Source Address.
  • If the PDU Format (PF) byte is < 240, the PGN uses only the PF byte (peer-to-peer addressing); the lower byte is the destination.
  • If PF ≥ 240, both PF and PS bytes form the PGN (broadcast addressing).

Example: CAN ID 0CF00400 → right-shift 8 → 0CF004 → PF = F0 (≥240) → PGN = F004 (EEC1 — Electronic Engine Controller 1).

SPN Decoding

Each PGN contains one or more Suspect Parameter Numbers (SPNs). Each SPN definition specifies:

  • Byte offset — which data byte(s) to read from the 8-byte payload.
  • Bit length — how many bits the value occupies.
  • Resolution & Offset — scaling formula: value = raw × resolution + offset.
  • Labels — for status/enumerated fields (e.g., 0 = Off, 1 = On).

Example: SPN 190 (Engine Speed) in PGN F004 uses bytes 3–4, resolution 0.125 RPM/bit, offset 0. If bytes 3–4 contain 0x1388 (5000 decimal), RPM = 5000 × 0.125 = 625 RPM.

PGN Library

CAN Insight PRO includes a built-in library of 64+ J1939 PGNs covering engine, transmission, braking, fuel, electrical, and diagnostic parameters. Each PGN definition maps its SPNs with byte positions, scaling factors, units, and human-readable labels.

🔋 SPN 5464 — Battery Charge Calculation Tutorial

What is SPN 5464?

SPN 5464 is the "Hybrid Battery Pack Remaining Charge" parameter — it tells you how much charge is left in a hybrid or electric vehicle's battery, as a percentage.

  • Found in: PGN FCB7 (Vehicle Electrical Power #4 — VEP4)
  • Units: Percentage (%)
  • Range: 0% to 160.6375%
  • Update rate: Every 10 seconds
  • Why over 100%? The range goes above 100% to account for measurement variations and battery conditioning states

Step 1 — Find the CAN message

Look for messages with PGN FCB7 in your CAN log. Example from a real electric truck log:

"00:02:20.302","18FCB7FC","Ext","","40 9C FF FF FF FF FF FF"

Breaking this down: 18 = priority, FCB7 = PGN, FC = source address, 40 9C FF FF FF FF FF FF = data bytes.

Step 2 — Extract bytes

SPN 5464 uses bytes 0 and 1 of the data payload. From 40 9C FF FF FF FF FF FF we need: 40 (byte 0) and 9C (byte 1).

Step 3 — Swap byte order (little-endian)

CAN messages use little-endian byte order — the least significant byte comes first. Swap: 40 9C → 9C40.

Step 4 — Convert hex to decimal

9C40 hex = (9×4096) + (12×256) + (4×16) + 0 = 36,864 + 3,072 + 64 = 40,000

Step 5 — Apply resolution

SPN 5464 resolution = 0.0025% per bit. So: 40,000 × 0.0025 = 100% ✅

Quick reference formula

Battery Charge (%) = (Byte₀ + Byte₁ × 256) × 0.0025

More examples

  • B0 9A → 9AB0 → 39,600 → 39,600 × 0.0025 = 99.0%
  • 20 4E → 4E20 → 20,000 → 20,000 × 0.0025 = 50.0%
  • 88 13 → 1388 → 5,000 → 5,000 × 0.0025 = 12.5%
  • FF FF → 65,535 → Not Available (vehicle off or unsupported)

Common questions

Why is the byte order backwards? It's like different countries writing dates differently (MM/DD vs DD/MM). CAN bus uses little-endian, so we swap the bytes.

What if I get over 100%? Valid — can happen during regenerative braking or battery conditioning. Max is 160.6%.

What does FF FF mean? "Not Available" — the vehicle isn't providing this data (ignition off, sensor error, or feature unsupported).

🎉 Fun CAN Bus Facts

  • 🚗 CAN was invented by Robert Bosch GmbH in 1986 — originally for reducing wiring weight in Mercedes-Benz cars. A modern truck can have over 2 km of wiring!
  • 📡 A single CAN bus can carry up to 2,000+ messages per second at 250 kbit/s. At highway speed, your truck is sending more data than a dial-up internet connection.
  • 🔢 J1939 defines over 8,500 SPNs — CAN Insight PRO decodes 200+ of the most important ones. That's the top 2.4% of all possible parameters!
  • 🌍 The FMS (Fleet Management System) standard is used by all major European truck manufacturers — DAF, Iveco, MAN, Mercedes-Benz, Renault Trucks, Scania, and Volvo all agreed on a common interface.
  • ⚡ Electric truck CAN data includes battery charge (SPN 5464) updated every 10 seconds. On a 300 kWh battery, each 1% = enough energy to power a house for 3 hours!
  • 🔧 The "Suspect Parameter Number" (SPN) naming comes from engine diagnostics — every parameter is a "suspect" when something goes wrong. It's like a detective lineup for your engine!
  • 📏 A standard CAN frame carries only 8 bytes of data — that's fewer characters than "Hello!!!" — yet it encodes everything from engine speed to battery voltage.
  • 🏎️ CAN bus is also used in Formula 1 cars, aircraft, medical devices, elevators, and even some coffee machines!
